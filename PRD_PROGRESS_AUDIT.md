- Sprint bucket 5: wire passenger and driver web-app surfaces to real backend API/state. UI scaffolding now exists and simulates lifecycle transitions locally; next step is server-authoritative booking/offer/trip wiring.
- Sprint bucket 6: repair the automated test suite by removing or rewriting stale auth tests and replacing them with PRD-aligned coverage for admin auth, fare rules, CRUD permissions, bookings, and state transitions.
